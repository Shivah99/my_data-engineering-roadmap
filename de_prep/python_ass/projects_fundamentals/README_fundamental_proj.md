PROJECT STRUCTURE

Each project is implemented as a standalone Python file.

Recommended structure:

```
project_root/
|
|-- personal_finance_calculator.py
|-- number_analyzer.py
|-- geometry_utility.py
|-- billing_system.py
|-- student_evaluation_system.py
|-- character_inspector.py
|-- README.txt
```
==================================================

HOW TO RUN THE PROJECTS

1. Ensure Python 3 is installed on your system
2. Open a terminal or command prompt
3. Navigate to the project directory
4. Run any project using:

   python filename.py

Example:
   python personal_finance_calculator.py

PROJECT LIST AND DESCRIPTIONS

--------------------------------------------------
PROJECT 1 – Personal Finance Calculator
--------------------------------------------------

Concepts Used:
input()
type casting
arithmetic operators
formatted output

Description:
A financial utility that performs multiple related calculations in one program.

Requirements:
- Take principal amount, time period, and rate of interest as input
- Calculate and display:
  simple interest
  average monthly interest
  cost price per item based on total profit.

--------------------------------------------------
PROJECT 2 – Number Analyzer
--------------------------------------------------

Concepts Used:
arithmetic operators
integer division
modulo operator
variable reassignment

Description:
A numeric analysis program using arithmetic logic instead of string manipulation.

Requirements:
- Take an integer input from the user
- Display:
  last digit of the number
  number without its sign
  reversed number using numeric logic only
- Swap two numbers provided by the user

--------------------------------------------------
PROJECT 3 – Geometry Utility
--------------------------------------------------

Concepts Used:
arithmetic operators
expressions
reuse of variables

Description:
A reusable geometry calculator based on user-selected shapes.

Requirements:
- Allow the user to choose:
  rectangle
  square
  circle
- Calculate and display:
  area
  perimeter or circumference

Why This Matters:
- Demonstrates parameterized logic
- Avoids repetition and hardcoding
- Mimics real utility programs

--------------------------------------------------
PROJECT 4 – Billing System
--------------------------------------------------

Concepts Used:
if / elif / else
comparison operators
logical operators

Description:
A slab-based billing engine that calculates charges based on usage.

Requirements:
- Take usage input from the user
- Apply slab-based pricing rules
- Calculate the final bill amount
- Display a breakdown of the calculation

Why This Matters:
- Mirrors real-world billing systems
- Uses conditional logic with ranges
- Common in utility and service platforms

--------------------------------------------------
PROJECT 5 – Student Evaluation System
--------------------------------------------------

Concepts Used:
logical operators
conditional statements
comparison operators

Description:
A rule-based system to evaluate student performance.

Requirements:
- Take marks as input
- Validate the input values
- Calculate percentage
- Assign grades based on rules
- Display final evaluation result

Why This Matters:
- Represents decision-making systems
- Demonstrates validation and branching
- Widely used in academic platforms

--------------------------------------------------
PROJECT 6 – Character Inspector
--------------------------------------------------

Concepts Used:
strings
membership operator
character inspection logic

Description:
A character analysis tool for basic text preprocessing.

Requirements:
- Take a single character as input
- Determine:
  whether it is an alphabet or not
  whether it is a vowel or consonant
- Convert uppercase to lowercase and vice versa

Why This Matters:
- Useful for input sanitization
- Common in text processing pipelines
- Foundational for handling user input
